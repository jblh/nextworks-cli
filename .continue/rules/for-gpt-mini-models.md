description: file editing
alwaysApply: true
---

Use file edit tools only for code changes; do not use terminal commands for editing files.

When editing a file, read the latest file first, then make one carefully planned edit set.

Avoid multi_edit with overlapping or dependent replacements; split into separate edit calls if needed.

If an edit fails, re-read the file before retrying instead of improvising with terminal commands.

Do not switch tools unless the requested task actually requires terminal execution.
